Why it matters

Skill 市场的问题。想找的找不到，找到的不敢用。

description 过度泛化、下载量和星标失真，让 Agent 很难把真实需求匹配到真正可用的 Skill。meyo-skill-finder 要解决的是发现和信任两件事。

01

想找的找不到

创作者为了覆盖更多搜索，把描述写得宽泛抽象。真实任务进来时，关键词对不上、语义太散，真正适配的 Skill 被埋在大量候选里。

02

找到的不敢用

下载量、星标和文档评分不能证明它真的跑通过。边界、依赖、结果质量都不透明，Agent 找到一个 Skill 后仍然不知道能不能放心调用。

How Meyo Works

把 Skill 的“自我介绍”，换成可检索的真实战绩

meyo-skill-finder 同时生产评测内容和匹配链路：先把全网 Skill 放进真实任务里跑，再用 AI Search 把 Agent 的自然语言需求接到这些战绩上。

1

全网 Skill 获取

抓取技能内容，建立可评测候选池，避免只依赖单个平台的热度排序。

2

社区百万技能实测贴

把社区里真实任务、真实运行结果和横评内容沉淀成可检索的技能战绩。

3

多路检索召回

同时从 Skill 能力、社区实测贴和真实产出里召回，再把需求接到可验证的战绩上。

典型 Query：把产品卖点改写成小红书种草笔记，并生成 5 个标题

meyo-skill-finder 会从内容风格、平台语气和历史实测产出里召回，优先推荐在真实内容任务中表现稳定的 Skill。

✓ XHS Content Studio 平台语气更贴合
✓ Hook Title Generator 标题点击感更强
✓ Brand Tone Polisher 卖点表达更稳定
